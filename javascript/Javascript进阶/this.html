# this

  • 从某些角度来说，开发中如果没有this，很多的问题我们也是有解决的方案，但是没有this，会让我们编写代码变得非常的不方便
  • this在大多数情况下，this都是出现在函数中

# this在全局作用域指向什么

  1. 浏览器：window(globalObject)
  2. Node环境：{} (指向一个空对象)

# this到底指向什么呢？

    1. 函数在调用时，js会默认给this绑定一个值
    1. this的绑定和定位的位置(编写的位置)没有关系
    1. this的绑定和调用方式以及调用的位置有关系
    1. this是在运行时被绑定的

# this的绑定规则

  • 隐士绑定

    • 隐士绑定有一个前提条件
      1. 必须在调用的对象内部有一个对函数的引用(比如一个属性)
      2. 如果没有这样的引用，在进行调用时，会报找不到函数的错误
      3. 正是通过这个引用，简介的将this绑定到了这个对象上

  • 显示绑定

    • call-apply-bind
    • call和apply的区别
    • call和apply在执行函数时，是可以明确的绑定this的，这个绑定规则称之为显示绑定
    • bind会返回一个函数，可以直接调用返回的函数

  • new绑定

    • 使用new关键字来调用函数时，会执行如下操作

      1. 创建一个全新的对象
      2. 这个新对象会被执行prototype连接
      3. 这个新对象会绑定到函数调用的this上(this的绑定再这个步骤完成)
      4. 如果函数没有返回其他对象，表达式会返回这个新对象

    • 通过一个new关键字调用一个函数时，这个时候this实在调用这个构造器时创建出来的对象，this = 创建出来的对象，这个绑定过程就是new绑定

  • 绑定的优先级

    • new绑定 > 显示绑定(apply/call/bind) > 隐式绑定(obj.foo()) > 默认绑定(独立函数调用)
